ПОЧАТКОВІ КРОКИ:
1. перейти за наданим посиланням
2. завантажити zip-файл <>Code -> Download ZIP
3. розпакувати завантажений файл zip
4. відкрити папку Unconstrained-multivariable-optimization-master
5. використовуючи консоль викликати команду "python main.py" або запустити код у власному редакторі коду


У ВИПАДКУ ПОМИЛКИ ПРИ ЗАПУСКУ:
1. перевірити наявність Python версії 3.10+
2. перевірити наявність бібліотек: customtkinter, matplotlib, pandas, numpy
3. якщо програма надалі працює некоректно -> оновити бібліотеки до останньої версії


МОЖЛИВІ ПОМИЛКИ ПРИ РОБОТІ З ДОДАТКОМ:
-  некоректне введення даних:
   1. метод оптимізації повинен бути обраний серед доступних варіантів
   2. функція мети, градієнт, Гессе повинні мати формули - синтаксис яких підходить для Python-функцій
      приклади:
         5*x[0]**2 + 4*sqrt(5)*x[1]**2 - 13*x[0]*x[1] - 31 ,
         -2. * sin(.01 * x[0]) + 2. * cos(10. * x[1]) + 20.0 .
   3. початкова точки, точність повинні мати раціональні константи - синтаксис яких підходить для Python-значень
      приклади: 0 , 0.0 , .0 , 0.
-  великі розрахунки:
   1. якщо оптимізація буде мати велику кількість ітерацій - програма фізично не зможе віобразити всі ітерації алгоритму в таблиці й всі маршрути на графіку
      для цього випадку програма має параметр максимальної кількості ітерацій - стандартне значення якої є 100


ОСОБЛИВІ ВИПАДКИ:
-  у випадку появи віконця з повідомлення про помилку додатку - натиснути на іконку помилки або на крестик для видалення віконця
-  у випадку роботи з методами оптимізації програма потребує ввести дані, непотрібні для конкретного алгоритму - ввести порожні значення 0, 1 або будь-яке інше значення
-  у випадку маштабування графіку зникає відображення рівнів функції мети - налаштувати правий повзунок, який відповідає за інтервали функції за осью z
-  у випадку перемикання теми або кольорової мапи виникає некоректне фарбування елементів інтерфейсу - треба натиснути на зелену кнопку recover зверху-зліва для оновлення інтерфейсу
